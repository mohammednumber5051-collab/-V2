## 🟠 التحذيرات الهامة (5)

1. **عدم اتساق الخصومات** - استخدام `total - discount` في بعض الأماكن و `total - paid` في أخرى
2. **الفواتير الآجلة** - حالة الفاتورة قد تتغير بدون سبب واضح
3. **عدم كفاية التحقق من الأرصدة** - قد يحدث رصيد سالب في الصناديق
4. **عدم توثيق العلاقات** - لا يوجد ربط واضح بين الفواتير والمعاملات
5. **عدم معالجة المعاملات المرجعة** - الحذف قد لا يعكس التأثير بشكل صحيح

## 🛠️ خطوات الإصلاح

### المرحلة 1: التقييم (30 دقيقة)
```
1. اقرأ ACCOUNTING_AUDIT_REPORT.md كاملاً
2. فهم المشاكل الثمان الرئيسية
3. تقبل أن المنطق المحاسبي معاكس تماماً
```

### المرحلة 2: التخطيط (1 ساعة)
```
1. حدد الملفات التي تحتاج إصلاح
2. رتب الأولويات (الأولوية 1 ثم الأولوية 2)
3. أنشئ خطة اختبار شاملة
```

### المرحلة 3: الإصلاح (4-6 ساعات)
```
1. توحيد حساب التأثير (دالة واحدة)
2. إصلاح Debit/Credit
3. إصلاح أرصدة الشركاء
4. إصلاح حساب الأرصدة
5. إزالة التحديثات المضاعفة
```

### المرحلة 4: الاختبار (3-4 ساعات)
```
1. اختبر كل سيناريو محاسبي
2. تحقق من التوازن
3. استخدم FINANCIAL_AUDIT_CHECKLIST.md
4. وثق النتائج
```

### المرحلة 5: النشر (1 ساعة)
```
1. أرشفة نسخة من البيانات الحالية
2. تطبيق الإصلاحات تدريجياً
3. مراقبة الأرصدة
4. توثيق التغييرات
```
